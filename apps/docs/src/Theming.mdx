# Theming & Customization Guide

Welcome to the comprehensive theming guide for **Nova UI Kit**. This document provides everything you need to customize the look, feel, and behavior of components to perfectly match your brand and requirements. üé®

Nova's theming system is designed with **simplicity** and **maximum flexibility** in mind, giving you complete control over every aspect of your Web3 application's appearance.

-----

## üéØ Core Philosophy

Nova's theming system is built on three foundational principles:

1.  **CSS-First Approach:** All styling is controlled through CSS Custom Properties (variables), requiring no JavaScript configuration files or complex build processes.

2.  **Tailwind CSS v4 Integration:** Seamless compatibility with modern Tailwind workflows, allowing you to use design tokens directly as arbitrary values.

3.  **Layered Customization:** Multiple levels of customization‚Äîfrom simple variable overrides to complete component replacement‚Äîgiving you exactly the control you need.

**Why This Approach?**

Traditional theming systems often require complex JavaScript configurations, vendor lock-in, or limited customization options. Nova UI Kit solves this by exposing every visual aspect through standard CSS variables that integrate perfectly with modern CSS frameworks.

## üöÄ Quick Start: Theme Customization

### Step 1: Setup Your CSS Structure

Ensure you have the correct import order in your main stylesheet:

```css
/* src/styles/app.css or src/styles/globals.css */

/* 1. Import Nova foundation (REQUIRED) */
@import '@tuwaio/nova-core/dist/index.css';

/* 2. Import Nova feature packages (as needed) */
@import '@tuwaio/nova-connect/dist/index.css';
@import '@tuwaio/nova-transactions/dist/index.css';

/* 3. Import Tailwind CSS */
@import 'tailwindcss';

/* 4. Your custom theme overrides go here */
:root {
  /* Your custom variables */
}
```

### Step 2: Override Design Tokens

Add your brand colors and customizations after the imports:

```css
/* Your custom theme overrides */
:root {
  /* Brand Identity */
  --tuwa-color-primary: #6366f1; /* Indigo-500 */
  --tuwa-color-primary-hover: #4f46e5; /* Indigo-600 */

  /* Semantic Colors */
  --tuwa-success: #10b981; /* Emerald-500 */
  --tuwa-error: #ef4444; /* Red-500 */
  --tuwa-warning: #f59e0b; /* Amber-500 */

  /* Custom Gradients */
  --tuwa-gradient-primary: linear-gradient(135deg, #6366f1 0%, #8b5cf6 100%);
}

/* Dark mode variations */
.dark {
  --tuwa-color-primary: #818cf8; /* Indigo-400 */
  --tuwa-success: #34d399; /* Emerald-400 */
  --tuwa-error: #f87171; /* Red-400 */
}
```

### Step 3: Test Your Changes

Your customizations will automatically apply to all Nova components. Test different components to ensure your theme works across all use cases.

### Nova Connect Customization

#### Behavioral Customization

```tsx
// Comprehensive Nova Connect Provider configuration

  // Visual Options
  appChains={appEVMChains}

  // Internationalization

    connectWallet: 'Connect Wallet',
    disconnect: 'Disconnect',
    connecting: 'Connecting...',
    connected: 'Connected',
    wrongNetwork: 'Wrong Network',
    switchNetwork: 'Switch Network',

  }}

  // Advanced Customization
  modalProps={{
    showRecentTransactions: true,
    showNetworkSwitcher: true,

  }}

  // Component Replacement

    ConnectButton: CustomConnectButton,

  }}

  {/* Your app */}

```

#### Custom Connect Button Variants

```tsx
// Create custom styled variants
import { cn } from '@tuwaio/nova-core';
import { ConnectButton } from '@tuwaio/nova-connect/components';

function CustomConnectButton({ className, ...props }: ConnectButtonProps) {
  return (
    <ConnectButton
      className={cn(
        // Override base styles
        'bg-gradient-to-r from-purple-600 to-blue-600',
        'hover:from-purple-700 hover:to-blue-700',
        'text-white font-semibold',
        'px-6 py-3 rounded-xl',
        'shadow-lg hover:shadow-xl',
        'transition-all duration-200',
        className
      )}
      {...props}
    />
  );
}
```

#### Custom Transaction Components

```tsx
// Custom transaction status component
import { cn } from '@tuwaio/nova-core';
import { TransactionStatus } from '@tuwaio/nova-transactions';

function CustomTransactionStatus({
  status,
  hash,
  className
}: { status: TransactionStatus, hash: string, className: string }) {
  return (
    <div className={cn(
      'flex items-center gap-3 p-4 rounded-lg',
      'border border-[var(--tuwa-border-primary)]',
      {
        'bg-[var(--tuwa-success-bg)] border-[var(--tuwa-success-accent)]': status === 'success',
        'bg-[var(--tuwa-error-bg)] border-[var(--tuwa-error-accent)]': status === 'failed',
        'bg-[var(--tuwa-pending-bg)] border-[var(--tuwa-pending-accent)]': status === 'pending',
      },
      className
    )}>
      <TransactionStatusIcon status={status} />
      <div className="flex flex-col">
        <span className="font-medium">
          {status === 'success' && 'Success'}
          {status === 'failed' && 'Failed'}
          {status === 'pending' && 'Pending...'}
        </span>
        {hash && (
          <a
            href={`https://etherscan.io/tx/${hash}`}
            target="_blank"
            rel="noopener noreferrer"
            className="text-sm text-[var(--tuwa-text-accent)] hover:underline"
          >
            {hash.slice(0, 8)}...{hash.slice(-6)}
          </a>
        )}
      </div>
    </div>
  );
}
```
